創站動機


源於想展現我Python學習的成果

但現在想想以網頁來展現Python好像也挺奇怪的(?

當時決定時真的是一時興起

想說既然之前有學過html，何不把它拿出來用?

於是整個計劃就誕生了w

一方面是想在高中結束前做個大計畫，另一方面是想挑戰自己對於程式的能力

製作的過程可謂十分艱苦QQ，且聽我依時間娓娓道來，一方面也為本篇的心得



1.Html,JavaScript,CSS忘光光，善用工具是上策


這件事老實說十分致命，對於要架網站這回事這三項基本不可或缺，要在我差不多快忘光的領域上重新學習屬實有些克難

於是在搜索了許多資料後，我終於找到了黑暗中的明燈--Bootstrap!

Bootstrap

Bootstrap提供了許多元件的模板(如首頁的字卡、作品列表的手風琴...)，讓我無需土炮各種css,style,animation就能輕鬆排出我想要的排版，雖然需要具備基本的除錯能力啦。

打個比方，在製作作品列表時遇到了按鈕沒對應到彈出視窗的問題

才發現原來要對應到id才能成功，不然會變成點下方的按鈕卻彈出上一個按鈕應該出現的東西的錯誤

(框起來的地方需對應到)




2.網站上線問題的過程，更加確信了AI是我未來的目標


網站架構稍微完整一點後，就要開始考慮網站上線的問題了

那時候選用Google Cloud Platform(GCP)為雲端主機時沒有想太多，去找資料後才發現大多數教學品質都參差不齊、中間許多缺漏和版本太舊的情形時常有

基於我沒有Linux指令和網站相關知識的情況下，我選擇求助於科技--ChatGPT

AI可謂是網站成功架設的一代功臣啊!

這或許也是為甚麼我希望朝向Machine Learning的方向前進的緣故:D


可他總不是萬能的，還是有些問題困住我無法向前

就是防火牆...

在缺少GCP網路資源的狀況下我無法解決防火牆的問題，於是我轉而求助正在就讀資工系的表哥，並了解到了我需要打開程式監聽之port的防火牆

譬如我開在8000port，我就必須在GCP的網頁中建立允許8000port的規則(依此類推)




3.魔鬼藏在細節裡，注意細節，避免損失


開啟8000port後，我和表哥決定採取防火牆的第三方套件”ufw”

安裝完成後一樣先用ufw的指令允許8000port後，再打開ufw:

sudo ufw allow 8000/tcp
sudo ufw enable


在測試成功後，就興奮地去吃晚餐了

結果...接下來不管怎麼連都連不進SSH了


在網路上搜尋了一會兒後，總算有比較詳細的解法了

追根究底的原因只有一個:忘了輸入一行程式碼

sudo ufw allow ssh

......

因為少這行程式碼，我必須去編輯這台VM:

啟用序列控制台、設定後臺帳密......等

GCP 虛擬主機連線失敗救援教學―第三方軟體阻斷 SSH 連線

嘗試後卡在了帳密過不去

再三考量後決定打掉重來

幸好資料並不多才能這樣做

也算順便給了我梳理整個流程的機會......




4.從失敗裡學習，成就最後的成功的果實


在經歷了無數報錯和修復中，最後一次的意外讓我能從頭梳理整個過程

我統整了建立VM後應該要設定的、輸入的指令

在最後的測試中，快速且順利的成功建立起這個網站

那種成就感到現在還難以忘懷、難以言喻，如同兩個月陰霾後的曙光

告訴著我這一路上的努力就是為了此時此刻

對於他人可能只是個陽春的網站

但對我來說這是我寫程式路上的里程碑
